Project Overview

An interactive summary of the design brief for the AstroKundali application.

Primary Goal

To provide a simple, accurate, and visually appealing platform for users to generate, view, and manage their Vedic astrology birth charts. This brief outlines the design and functional blueprint to achieve this goal.

Target Audience

  • Individuals in India and abroad interested in their personal astrology.
  • Families creating charts for newborns or for matchmaking.
  • Amateur astrology students and enthusiasts.

Core Features

Key functionality defined by user stories. Click each user type to see their goals.

  • I want to quickly generate a Kundali without creating an account (Guest Mode).
  • I want to create an account to save my profiles and access them later.
  • I want to see a dashboard of all my saved profiles (e.g., "Me," "My Son," "My Partner").
  • I want to create a new Kundali profile by entering a Name, Date of Birth, Time of Birth, and Place of Birth.
  • I want to easily find the Place of Birth using a search-friendly map/city finder (essential for lat/long).
  • I want to see the main Lagna (Ascendant) chart clearly.
  • I want to toggle between different chart styles (North Indian, South Indian, East Indian).
  • I want to view basic planetary details (Planet, Rashi, Degree, Nakshatra).
  • I want to check my Vimshottari Dasha (major planetary periods).
  • I want to see basic Panchang details for my birth (Tithi, Vara, Nakshatra, Yoga, Karana).
  • I want to download a simple PDF summary of my chart.

Screen-by-Screen Flow

The detailed blueprint for each application screen. Click a screen to view its components.

Screen 1: Splash & Onboarding

Purpose:

Initial loading and branding screen.

Components:

  • App Logo (e.g., a stylized constellation or lotus).
  • App Name & Tagline (e.g., "AstroKundali: Your cosmic map").
  • Simple animation or graphic.

Flow:

Automatically transitions to the Home/Login screen after 2-3 seconds.

Screen 2: Welcome / Home (Guest)

Purpose:

The main entry point for new users.

Components:

  • Hero Section: A beautiful, calming astrological graphic.
  • Primary CTA: A prominent button: "Create New Kundali".
  • Secondary CTA: "Login / Sign Up" (less prominent, perhaps top-right).
  • Value Proposition: "Enter your birth details to discover your complete Vedic birth chart."

Screen 3: Create Kundali Form

Purpose:

To collect the necessary data with high accuracy.

Components:

  • Form Title: "Create Your Kundali"
  • Input Field: "Full Name"
  • Input Field: "Date of Birth" (Use a native calendar/date picker).
  • Input Field: "Time of Birth" (Use a native time picker).
  • Input Field: "Place of Birth (City, Country)"
  • CRITICAL: This must be powered by a Google Places API (or similar) to autocomplete and fetch the exact latitude and longitude automatically.
  • Toggle: "Save this profile to my account?" (Appears if logged in).
  • Primary CTA: "Generate Kundali" (This button should have a loading state).

Screen 4: Kundali View (Main Report)

Purpose:

To display the generated Kundali in a clear, tabbed interface.

Components:

  • Header: Shows the Profile Name, DoB, ToB, and PoB.
  • Sub-Header (Tabs): Core navigation: `Chart` | `Planets` | `Dasha` | `Details`
  • Settings Icon (Top Right):
    • Allows user to change Chart Style (North Indian, South Indian).
    • "Download PDF" option.
  • Tab 1: Chart (Default): Main Lagna chart, with toggle for Navamsa (D9).
  • Tab 2: Planets: A clean table (Planet | Rashi | House | Degree | Nakshatra | Pada).
  • Tab 3: Dasha: Vimshottari Dasha (expandable list of Maha/Antar/Pratyantardasha).
  • Tab 4: Details: Basic Panchang info (Tithi, Vara, Nakshatra, Yoga, Karana).

Screen 5: My Profiles (Logged-in Home)

Purpose:

The dashboard for a logged-in user.

Components:

  • Header: "My Kundalis"
  • Primary CTA: "Add New Profile" (Floating Action Button in the corner).
  • List: A card-based list of all saved profiles.
  • Each Card: Name, DoB & ToB, "View Kundali >" link.

Screen 6: Login / Settings

Purpose:

Account management.

Components:

  • Login Options: Email/Password, Sign in with Google, Sign in with Apple.
  • Logged-in View (Settings):
    • "Edit Profile"
    • "Default Chart Style" (North/South)
    • "Language"
    • "Logout"
